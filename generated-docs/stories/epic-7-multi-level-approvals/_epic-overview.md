# Epic 7: Multi-Level Approvals

## Description

This epic implements the three-level approval workflow (L1, L2, L3) with approval/rejection capabilities. Rejection at any level returns the workflow to Data Preparation phase. L3 rejections require a mandatory rejection reason.

## Stories

1. **Level 1 Approval Screen** - Initial data completeness review
   - File: `story-1-level-1-approval.md`
   - Status: Pending

2. **Level 2 Approval Screen** - Portfolio confirmation and risk checks
   - File: `story-2-level-2-approval.md`
   - Status: Pending

3. **Level 3 Approval Screen** - Final sign-off with mandatory rejection reason
   - File: `story-3-level-3-approval.md`
   - Status: Pending

4. **Approval Workflow State Management** - Handle state transitions and access control
   - File: `story-4-approval-workflow-state.md`
   - Status: Pending
